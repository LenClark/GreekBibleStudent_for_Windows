1 General Information about Greek Bible Student (GBS)

The aim of Greek Bible Student is to provide you with a version of the Greek New Testament and the Septuagint with some tools for exploring it on your own computer, This means that you can use it even when the internet is not available. It also means that you can work at your computer's speed and not that of your internet connection. (Please see the final section, Caveats and Compliments, for a brief explanation of sources and acknowledgements.)

We'll introduce you to the different parts of the application in detail later but let's get an overview now of what Bible Reader offers. It provides

  • The text of both New Testament and the Septuagint, displayed for a selected chapter
  • A parse window: simply by clicking and "Analysing" a word, you can get a grammatical analysis of that word;
  • At the same time, it will also try to give you details of how the word was used (as provided by the Liddell & Scott Intermediate lexicon);
  • A search facility allows you to get a list of all the verse where the word is used, together with some additional sophistication in search criteria;
  • A notes area: a feature that allows you to create, save and restore notes for each verse in the New Testament;
  • A vocabulary utility that allows you to list words in a verse or chapter, categorising by part of speech.

So, let's look at the application in more detail.

3 The main Text Area

3.1 The Basics

It's pretty obvious what happens here.

The New Testament (NT) text and the Septuagint (LXX) text are kept separate and are accessed through separate controls. So, to read the NT text, click on the first tab at the top of the text pane and to read the LXX text click on the second:

If you are reading the NT, you can change the book you are reading by selecting from the list in the Book list, in the Orientation Panel. Similarly, you can change the chapter by selecting from the Chapter list. Note that, if you change the book, it will automatically reset to the first chapter of that book.

If you want to do the same for the LXX, you need to select the second tab - Septuagint (Old Testament) - and you will be presented with a very similar setup.

In both cases, you can also select a specific verse from the relevant pull-down list but there will be no visible change. (We will talk about the purpose of the list of verses later.)

If you have a note displayed for your current verse and you change this verse reference, that note will disappear from view. (Don't worry! It will be conserved.) Similarly, if the verse you change to already has a note, then this note will now be displayed. We'll talk about notes later.

3.2 Some Additional Options

If you right click anywhere on the text area, a small menu will pop up:

This gives you a number of options.

3.2.1 Copy Current Word to ...

Selecting this option will display a new dialog:

This dialog allows you to specify:

  1. whether to direct the copy to the clipboard (so that it can subsequently be pasted into the application of your choice) or to the currently active note;
  2. whether to include the reference in the copy or not; and
  3. whether to include accents in the copied text or not.

If you are doing quite a lot of copying, it could be annoying to be constantly presented with this dialog and have to check various radio buttons every time. So, we have also provided a checkbox that allows you to specify that any future copies of words will automatically have the same options - in which case, this dialog will not appear again. (We will explain later how you can cancel this automatic feature, so that you can specify different options.) In any case, these options will only apply to the current session of Greek Bible Student: when you next restart the application, you will be presented with this diaolog again, the first time you copy a word.

If you select Memory, this will copy the currently selected word to the clipboard. This means that you can paste it into another application, such as a Word Processor. (If you're not sure how to paste into your application, check its help facility, look on Google or ask a friend.)

When you right click to get the Context Menu, make sure that the mouse is over the specific result and word that you are interested in.

3.2.2 Copy Current Verse to ...

This operates in exactly the same way except that it will copy the whole verse rather than a specific word. Again, be careful that you position the mouse correctly. The first time you select this in any session you will be presented with the same dialog. This time, it will specifically relate to copying the verse rather than a specific word.

3.2.3 Copy Current Chapter to ...

Predictably, this does the same thing but for the whole chapter. This time, the position of the mouse is relatively unimportant. Once again, you will be able to select the same options from the dialog.

3.2.4 Copy Selected Text to ...

You can select a small amount of text by using a click and drag technique: click the left mouse of the button, as normal, at the place where you want to selection to start but hold the button down rather than letting go; then, while holding the button down, drag the mouse until it is over the place where you want the selection to finish; then let go. Once you have done this, you can right click anywhere in the text area and select the menu option to copy the selected text. As before, the same options are available and can be set from the dialog.

3.2.5 Double Click - A Warning

Normally, with any text, you can select a word using a double click - i.e. if you click the left mouse button twice in succession, the entire word can be highlighted. This action can even be extended to multiple words by doing the "double click", holding the button down after the second click and then dragging the mouse across the text you want to select. Unfortunately, we haven't been able to make this work correctly in GBS: for some reason, the Unicode script used for the Greek text wrongly reports the length of words and, occasionally, Windows thinks the word is longer than it is. This means, when you select a word by double clicking, it can highlight the beginning of the next word as well.

As a result, we counsel you to avoid attempting to make a selection using double clicks. If you do use the double click method, watch out for errors in the extent of the selection.

You can use this option to remove options that you previously selected. This menu option is greyed out if you haven't actually set any copy options yet. If you have set options for one or more copy menu options, the following dialog will be shown:

5 History

Greek Bible Student keeps a track of passages you have selected. If you want to return to a previous Book and Chapter, simply click on the little down arrow of the relevant History Combo Box:

Then select the previously-used passage that you want. Obviously, if you want to reuse a LXX passage, you will need to select it from the History of LXXX access list.

Although not strictlly history, you can also select the previous chapter by clicking on the left arrow as: in the top, orientation bar, or the next chapter by clicking on the right arrow. (Of course, if you are currently referencing a LXX passage, the title will say LXX Chapter.)

Note that the bottom right pane also includes a tab for Liddell & Scott Appendices. If you click on this, you will see a series of tabs on the bottom of the pane as well. Each of these will display information found in the original lexicon. There is no intelligence in these pages but they are they so that you can check abbreviations and the like that you might find in the Lexical Information.

6. Searching

If you click on the Search tab in the lower right pane, you will find a fairly busy pane which allows you to configure a search:

This will take quite a bit of unpacking and some time to get used to, but it is to be hoped that this will be a useful part of GBS.

Obviously, the word to search for must be entered into the first textbox: Search for the word. The question is: How do you type a Greek word into the box?

Probably the most useful mechanism requires a return to the text area. If you right click on a word in the text area, a context menu option allows you to copy that word into this box:

If you select this menu option it will insert the selected word into the box. (You can't see it here but the word μετοικεσίας was selected.) So, with the word in the textbox, simply click on the Perform Search button and GBS will look for all the occurrences in the NT and the LXX:

Notice the scroll bar to the right of the search results. This means that you can click on the light grey area below the "thumb" and scroll down a page for more results. Alternatively, you can "drag" the darker grey thumb (the small square) and drag down as much or as little as you want.

Notice also the message at the bottom of the results. This was created while it was performing the search and is a report on progress. Be patient: the application is scanning through a lot of text and can take a little while.

So,clicking on the same three groups with Omit selected will then remove the NT books to the list:

To the left of the book list is an option that we have ignored up to now: All words with the same root. We have been searching all words with the same root. This will not search for both verbs and cognate nouns but will search for all variations of conjugations of a verb or declinations of a noun or adjective. If you really only want occurrences of exactly the word you entered in the Search box, then click on the radio button, Exact match:

6.4 A second way of entering your search word

In general, I have found the method described above as the most common method by far. However, there is a second way. If you look on the bottom left of the application window, you will see the button: Show Keyboard:

Click on this button and a virtual keyboard will be revealed:

Anything you type will be entered into the textbox, Entered text. The radio buttons on the bottom will determine where the final word will go when you click on Use Word . The main benefit of this keyboard is that you can easily enter Greek text into your notes. However, you can also use this to populate your search option by selecting the Primary Search String radio button option. Now, when you type, the characters will be entered in the Search textbox.

Of course, you "type" using the mouse. These do not respond to keyboard presses. To get the accents, breathings iota subscript and the like, click on the relevant symbol first, then on the character that will bear it. If the addition is not allowed, it will simply be ignored. Note: if you're not sure of which key or symbol is which, hover the mouse over the key briefly and a "hint" will appear to tell you.

You may need to experiment with this option a little. You will also need to be careful that you have typed the word exactly.

6.5 Secondary Search Criteria

No doubt the hawk-eyed user has already noted several references to Secondary Searches and Secondary Search Strings. You can either redirect a second typed word by selecting the Secondary Search String radio button option on the virtual keyboard, or you can use another search option when you right click on a selected word in the text:

This will then modify the search area:

When you click on Perform Search you will now search for all occurrences of the primary word that occur within n words of the secondary word:

You will need to be very patient when using this option: the search process is a lot more complex and can take quite some time. If you perform a search that takes a long time, you should notice that a new button appears at the bottom of the setup area:

Clicking on this button will stop the search immediately. Some results found up to that point may be displayed.

You can vary the strictness or slackness of criteria by varying the "distance" of the two words, up to a separation of 10 words. It will be for you to judge how far the two words can be apart before there mutual occurrence becomes coincidental rather than significant.

Note also that this search can return matches across two or even three verses. (In fact, in extreme, it could be up to 5 (very short) verses. This explains why some results will look a bit ungainly and have two verses without a break between them. All the other utilities (root exact; excluding or including books or groups of books) work in the same way.

These search results are primarily for your information. They form a kind of concordance. However, you can also make use of them. Right click on any specific result and you will see the following Context Menu:

7. Vocabulary Lists

GBS allows you to create vocab lists for a specific chapter or verse. To do this, click on the Set up Vocabulary tab in the bottom half of the right hand pane:

You can set up various options much as described for the Search facility.

7.1 List vocab for ...

You can create a vocab list for either a single verse (the current verse) or for the whole chapter (the entire current chapter). If you opt for the verse, make sure that the verse you are interested in is correctly set in the correct pull-down list in the Orientation Panel at the top of the window. For both options, make sure that the correct text is active (NT or LXX).

7.2 Select specific types of words

This is not an either/or choice: you can check or uncheck as many grammatical categories as you want. So, for example, if you are only interested in Nouns, make sure that there is a tick in the Nouns box and all others are left blank. As another example, if you want both adjectives and adverbs but nothing else, then ensure that the two boxes, Adjectives and Adverbs are ticked and nothing else.

As the note beside the check boxes says, if you leave everything blank, then all words will be listed.

12 Additional Information about the Main Window

12.1 Getting the Window the way you want it

You can resize the window by dragging the window edges or corners in any desired direction. We will assume that you are familiar with resizing windows (and click-and-drag actions, in particular) and, if you are not, we recommend that you read a book on using your operating system.

The advantages of making the window bigger are:

  • The main text may be easier to read;
  • You will see more of the text if the chapter is a long one;
  • It may make many of the text boxes and buttons less cluttered and easier to use.

The advantages of a smaller window may be:

  • That the application better fits a small screen;
  • You may wish to have other applications open at the same time and this makes it easier to navigate around your estate.

What is important about the resizing of your window is that GBS will remember your settings and return the window to the same size next time you use the program.

The window is divided into two halves and, between the main text area and the collection of small panes on the right hand side, there is a Splitter Bar. Slowly move your mouse from the main text area across to the right and, as you cross the divider, the cursor shape should change:

At the point the curser has changed you can click and drag the bar with your mouse. The benefit of this change is that you can maximise the area in which you have main interest.

The application will remember the relative sizes of the two halves of the window as well when you return to the application.

14. Caveats and Compliments

The text of the NT used in this application seems to be based on the transliterated version developped by CCAT (the Center for Computer Analysis of Texts) and is (or, at least, was) part of the University of Pennsylvania. This seems to have been developed by the Society of Biblical Literature, Atlanta, Georgia (with links to Logos Software) and additional thanks go to James Tauber who has made many corrections to the original parsing of the Gk NT. It has to be said that the nature of computer analysis of the NT text has become quite extensive and it's difficult to be sure that adequate acknowledgements have been made, so this includes an apology if we have failed to give adequate credit where credit is due.

The text seems to be essentially that of Westcott and Hort (although, don't hold us to that) and this means that the text, as displayed, will vary from the latest versions produced by the United Bible Society (and the current version [version 28] is certainly recommended). In other words, this does not replace the need for a decent printed version of the Greek text.

The version of the LXX used also seems to have originated from CCAT (which, in turn, is based on the Rahlfs text) but some modifications were facilitated from a member of a Classical Greek forum and, sadly, we did not note his name or details at the time and are unable to make adequate acknowledgements. It is with sorrow that we admit this.

Because the path to the final versions of both NT and LXX have been long and convoluted, there is every probability that the data contains errors. We apologise for this and counsel caution in using the application: where conclusions are significant, check your facts carefully. But, then, you should do this anyway! Despite these words of caution, we think the benefits of using the software still outweigh the problems.
